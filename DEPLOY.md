# Vercel Deployment Checklist

This document outlines the final steps to launch FitLook on Vercel. All code-level hardening, config creation, and security patches have been successfully applied to the codebase.

## 1. Environment Variables
You MUST add the following environment variables in the Vercel Dashboard (**Project Settings -> Environment Variables**) before deploying:

### Supabase
- `NEXT_PUBLIC_SUPABASE_URL` (e.g. `https://nwpdahypzesoojsgbfpa.supabase.co`)
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY` (Required for Razorpay webhooks and admin ops)

### AI Models (Hugging Face / Nvidia)
- `HUGGINGFACE_API_TOKEN`
- `NVIDIA_API_KEY`

### Razorpay Payments
- `NEXT_PUBLIC_RAZORPAY_KEY_ID`
- `RAZORPAY_KEY_SECRET`
- `RAZORPAY_WEBHOOK_SECRET`

## 2. Deployment Steps
- [ ] Connect your GitHub repository to Vercel.
- [ ] Vercel will automatically detect the **Next.js** framework preset.
- [ ] The **Build Command** should automatically be `npm run build` or `next build`.
- [ ] The **Output Directory** should automatically be `.next`.
- [ ] Add the environment variables listed above.
- [ ] Click **Deploy**.

- [ ] **Preview Deployments:** Verify that preview deployments work when you open a PR.
- [ ] **Custom Domain:** Configure your custom domain (e.g. `fitlook.ai`) in Project Settings -> Domains.
- [ ] **Analytics:** Enable Vercel Web Analytics in the Vercel dashboard to track visitor metrics.
- [ ] **Razorpay Webhooks:** Ensure your Razorpay dashboard points to your new Vercel production URL for webhooks (`https://your-domain.com/api/webhooks/razorpay`).
